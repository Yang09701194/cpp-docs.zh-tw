# ATL 專案精靈
[!INCLUDE[vs2017banner](../../assembler/inline/includes/vs2017banner.md)]

Active Template Library \(ATL\) 是一組樣板 \(Template\) 架構的 C\+\+ 類別，能簡化寫入小型和快速的 COM 物件。  ATL 專案精靈建立具有包含 COM 物件結構的專案。  
  
## 概觀  
 這個精靈頁面將會描述目前您建立之 [ATL 專案的應用程式設定](../../atl/reference/application-settings-atl-project-wizard.md)。  依預設，專案具有下列設定：  
  
-   動態連結程式庫指定伺服器為 DLL 和 \(同處理序伺服程式。  
  
-   屬性會指定您的專案使用屬性。  
  
 若要變更這些預設值，請按一下精靈左欄中的 \[**應用程式設定**\] 並在 ATL 專案精靈的頁面中進行變更。  
  
 如需預設專案設定 \(包括字元集 \(Character Set\) 選擇、預設連結\) 的詳細資訊，請參閱[預設的 ATL 專案組態](../../atl/reference/default-atl-project-configurations.md)。  
  
 在建立 ATL 專案之後，您可使用 Visual C\+\+ [程式碼精靈](../../ide/adding-functionality-with-code-wizards-cpp.md)將物件或控制項加入專案。  您可使用程式碼精靈將下列類型的增強功能加入至基本的 ATL 專案：  
  
-   [將物件和控制項加入至 ATL 專案](../../atl/reference/adding-objects-and-controls-to-an-atl-project.md)  
  
-   [將新介面加入至 ATL 專案](../../atl/reference/adding-a-new-interface-in-an-atl-project.md)  
  
-   [將 COM\+ 1.0 元件加入至 ATL 專案](../../atl/reference/adding-an-atl-com-plus-1-0-component.md)  
  
 除此之外，當您建立並增強 ATL 專案時，可考慮執行下列的工作：  
  
-   [讓 ATL 物件變成無法建立的](../../atl/reference/making-an-atl-object-noncreatable.md)  
  
-   [最佳化 ATL 專案的編譯器](../../atl/reference/specifying-compiler-optimization-for-an-atl-project.md)  
  
 您可以在[專案屬性](../../ide/general-property-page-project.md)頁面設定專案屬性 \(例如，[是否要靜態連結至 CRT](../../atl/programming-with-atl-and-c-run-time-code.md)\)，也可以為 ATL 專案設定[組建組態](../Topic/Understanding%20Build%20Configurations.md)。  如需詳細資訊，請參閱[使用專案來建置應用程式](http://msdn.microsoft.com/zh-tw/3339fa90-bac2-4b95-8361-662a2e0e7dfe)。
