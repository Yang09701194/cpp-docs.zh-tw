# <a name="c26167"></a>C26167

> 警告 C26167：可能會在函數 \<func > 中釋放解除未保留鎖定 \<鎖定 >。

警告 C26167 類似于警告[C26117](../code-quality/c26117.md) ，不同之處在于信心層級較低。 例如，函數可能包含注釋錯誤。

## <a name="example"></a>範例

下列程式碼會產生 C26167 和 C26110。

```cpp

typedef struct _DATA {
    CRITICAL_SECTION cs;
} DATA;

_Releases_lock_(p->cs) void Leave(DATA* p) {
    LeaveCriticalSection(&p->cs); // OK
}
void ReleaseUnheldLock(DATA* p) { // Warning C26167
    int i = 0;
    Leave(p); // Warning C26110
}
```

## <a name="example"></a>範例

下列程式碼將會更正這些警告。

```cpp

typedef struct _DATA {
    CRITICAL_SECTION cs;
} DATA;

_Releases_lock_(p->cs) void Leave(DATA* p) {
    LeaveCriticalSection( &p->cs );
}

void ReleaseUnheldLock( DATA* p ) {
    EnterCriticalSection( &p->cs );
    int i = 0;
    Leave(p);
}
```
