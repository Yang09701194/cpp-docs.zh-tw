# <a name="__w64"></a>__w64

這個 Microsoft 專有關鍵詞已經過時。 在早于 Visual Studio 2013 的 Visual Studio 版本中，這可讓您標記變數，因此當您使用[/Wp64](../build/reference/wp64-detect-64-bit-portability-issues.md)進行編譯時，編譯器會回報當您使用64位編譯器進行編譯時，將會報告的任何警告。

## <a name="syntax"></a>語法

> *輸入* **__w64** *識別碼*

### <a name="parameters"></a>參數

*type*<br/>
這三種類型的其中一種，可能會在程式碼從32位移植到64位編譯器時發生問題： **int**、 **long**或指標。

*identifier*<br/>
您建立的變數的識別項。

## <a name="remarks"></a>備註

> [!IMPORTANT]
>  [/Wp64](../build/reference/wp64-detect-64-bit-portability-issues.md)編譯器選項和 **__w64**關鍵字已于 Visual Studio 2010 和 Visual Studio 2013 中淘汰，並從 Visual Studio 2013 開始移除。 如果您在命令列上使用 `/Wp64` 編譯器選項，則編譯器會發出命令列警告 D9002。 **__W64**關鍵字會以無訊息方式略過。 請使用以64位平臺為目標的 Microsoft C++編譯器，而不是使用此選項和關鍵字來偵測64位可攜性問題。 如需詳細資訊，請參閱[設定適用于64位 x64 目標的視覺效果C++ ](../build/configuring-programs-for-64-bit-visual-cpp.md)。

在其上有 **__w64**的任何 typedef 必須是 x86 上的32位和 x64 上的64位。

若要使用早于 Visual Studio 2010 的 Microsoft C++編譯器版本來偵測可攜性問題，您應該在任何在32位和64位平臺之間變更大小的 typedef 上指定 **__w64**關鍵字。 針對任何這類類型， **__w64**只能出現在 typedef 的32位定義上。

為了與舊版相容，除非指定了編譯器選項[/za \(停用語言擴充功能）](../build/reference/za-ze-disable-language-extensions.md) ，否則 **_w64**是 **__w64**的同義字。

如果編譯未使用 `/Wp64`，則會忽略 **__w64**關鍵字。

如需移植至 64 位元的詳細資訊，請參閱下列主題：

- [MSVC 編譯器選項](../build/reference/compiler-options.md)

- [將32位程式碼移植到64位程式碼](../build/common-visual-cpp-64-bit-migration-issues.md)

- [針對 64 位元 x64 目標設定 Visual C++](../build/configuring-programs-for-64-bit-visual-cpp.md)

## <a name="example"></a>範例

```cpp
// __w64.cpp
// compile with: /W3 /Wp64
typedef int Int_32;
#ifdef _WIN64
typedef __int64 Int_Native;
#else
typedef int __w64 Int_Native;
#endif

int main() {
   Int_32 i0 = 5;
   Int_Native i1 = 10;
   i0 = i1;   // C4244 64-bit int assigned to 32-bit int

   // char __w64 c;  error, cannot use __w64 on char
}
```
