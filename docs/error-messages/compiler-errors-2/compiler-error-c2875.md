# <a name="compiler-error-c2875"></a>編譯器錯誤 C2875
using 宣告造成 'class::identifier' 的多重宣告  
  
 宣告會造成相同的項目定義了兩次。  
  
 下列範例會產生 C2875:  
  
```  
// C2875.cpp  
struct A {  
   void f(int*);  
};  
  
struct B {  
   void f(double*);  
};  
  
struct AB : A, B {  
   using A::f;  
   using A::f;   // C2875  
   using B::f;  
};  
```
