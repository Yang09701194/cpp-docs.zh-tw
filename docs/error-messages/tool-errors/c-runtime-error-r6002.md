# <a name="c-runtime-error-r6002"></a>C 執行階段錯誤 R6002

浮點支援未載入

未連結必要的浮點程式庫。

> [!NOTE]
> 如果您在執行應用程式時遇到此錯誤訊息，則應用程式已關閉，因為它有內部問題。 此錯誤有幾個可能的原因，但通常是因為應用程式的程式碼發生缺陷，或是嘗試執行未針對特定電腦處理器所建立的應用程式。
>
> 您可以嘗試進行下列步驟來修正這個錯誤：
>
> - 使用 [**控制台**] 中的 [**應用程式和功能**] 或 [**程式和功能**] 頁面修復或重新安裝程式。
> - 在 [**控制台**] 中選取 [軟體更新] **Windows Update** 。
> - 檢查應用程式的更新版本。 如果問題持續發生，請洽詢應用程式廠商。

**程式設計人員的資訊**

當浮點程式庫未連結時，您的應用程式可能會發生此錯誤。 檢查其中一個原因：

- `printf_s` 或 `scanf_s` 函式的格式字串包含浮點格式規格，且程式未包含任何浮點值或變數。 若要修正這個問題，請使用浮點引數對應至浮點格式規格，或在程式中的其他位置執行浮點指派。 這會導致載入浮點支援。

- 編譯器只會在必要時載入浮點支援，藉此將程式的大小降至最低。 編譯器無法偵測格式字串中的浮點運算或浮點格式規格，因此它不會載入必要的浮點常式。 若要修正此問題，請使用浮點格式規格，並提供浮點引數，或在程式中的其他位置執行浮點指派。 這會導致載入浮點支援。

- 在混合語言程式中，連結程式時，會在 FORTRAN 程式庫之前指定 C 程式庫。 重新連結並指定 C 程式庫（最後一個）。
