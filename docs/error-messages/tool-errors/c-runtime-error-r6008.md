# <a name="c-runtime-error-r6008"></a>C 執行階段錯誤 R6008

沒有足夠的空間可供引數

> [!NOTE]
> 如果您在執行應用程式時遇到此錯誤訊息，則應用程式已關閉，因為它有內部記憶體問題。 有幾個可能的原因會造成此錯誤，但通常是因為記憶體不足的情況、環境變數所佔用的記憶體太多，或程式中的 bug。
>
> 您可以嘗試進行下列步驟來修正這個錯誤：
>
> - 關閉其他正在執行的應用程式，或重新開機您的電腦以釋放記憶體。
> - 減少應用程式的命令列引數數目和大小。
> - 使用 [**控制台**] 中的 [**應用程式和功能**] 或 [**程式和功能**] 頁面修復或重新安裝程式。
> - 在 [**控制台**] 中選取 [軟體更新] **Windows Update** 。
> - 檢查應用程式的更新版本。 如果問題持續發生，請洽詢應用程式廠商。

**程式設計人員的資訊**

有足夠的記憶體可載入程式，但記憶體不足，無法建立**argv**陣列。 這可能是因為記憶體不足的情況，或異常的命令列或環境變數使用方式所造成。 請考慮下列其中一個解決方案：

- 增加程式可用的記憶體數量。

- 減少命令列引數的數目和大小。

- 藉由移除不必要的變數來減少環境大小。
