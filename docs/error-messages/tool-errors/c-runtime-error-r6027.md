# <a name="c-runtime-error-r6027"></a>C 執行階段錯誤 R6027

空間不足，無法進行 lowio 初始化

> [!NOTE]
> 如果您在執行應用程式時遇到此錯誤訊息，則應用程式已關閉，因為它有內部記憶體問題。 此錯誤有幾個可能的原因，但通常是因為記憶體不足的狀況所造成。 這也可能是因為應用程式中的 bug、它所使用的視覺C++程式庫損毀，或是由驅動程式所造成。
>
> 您可以嘗試進行下列步驟來修正這個錯誤：
>
> - 關閉其他正在執行的應用程式，或重新開機您的電腦以釋放記憶體。
> - 使用 [**控制台**] 中的 [**應用程式和功能**] 或 [**程式和功能**] 頁面修復或重新安裝程式。
> - 如果應用程式在最近一次安裝另一個應用程式或驅動程式之前就正常運作，請使用 [**控制台**] 中的 [**應用程式和功能**] 或 [**程式和功能**] 頁面移除新的應用程式或驅動程式，然後再次嘗試您的應用程式。
> - 使用 [**控制台**] 中的 [**應用程式和功能**] 或 [**程式和功能**] 頁面，修復或重新安裝C++ Microsoft Visual 可轉散發套件的所有複本。
> - 在 [**控制台**] 中選取 [軟體更新] **Windows Update** 。
> - 檢查應用程式的更新版本。 如果問題持續發生，請洽詢應用程式廠商。

**程式設計人員的資訊**

當沒有足夠的可用記憶體可用來初始化 C 執行時間中的低層級 i/o 支援時，就會發生此錯誤。 此錯誤通常會在應用程式啟動時發生。 確認您的應用程式和其所載入的驅動程式和 dll 未在啟動時損毀堆積。
