# 命令列錯誤 D8045
[!INCLUDE[vs2017banner](../../assembler/inline/includes/vs2017banner.md)]

無法以 \/clr 選項編譯 C 檔案 'file'  
  
 只有 C\+\+ 原始程式碼檔案才能傳遞至使用 **\/clr** 的編譯。使用 **\/TP** 將 .c 檔案編譯為 .cpp 檔案，請參閱 [\/Tc、\/Tp、\/TC、\/TP \(指定原始程式檔類型\)](../../build/reference/tc-tp-tc-tp-specify-source-file-type.md)，取得詳細資訊。  
  
 如需詳細資訊，請參閱[\/clr \(Common Language Runtime 編譯\)](../../build/reference/clr-common-language-runtime-compilation.md)。  
  
 如果您使用 Visual C\+\+ 編譯 ATL 應用程式，則可能會發生 D8045。  如需詳細資訊，請參閱[如何：移轉至 \/clr](../../dotnet/how-to-migrate-to-clr.md)。
