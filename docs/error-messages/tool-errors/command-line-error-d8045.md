# <a name="command-line-error-d8045"></a>命令列錯誤 D8045

無法使用/clr 選項編譯 C 檔案 ' file '

只有C++原始程式碼檔案可以傳遞至使用 **/clr**的編譯。  使用 **/tp**將 .c 檔案編譯為 .cpp 檔案;如需詳細資訊，請參閱[/tc、/tp、/tc、/tp （指定原始程式檔類型）](../../build/reference/tc-tp-tc-tp-specify-source-file-type.md) 。

如需詳細資訊，請參閱 [/clr (Common Language Runtime Compilation)](../../build/reference/clr-common-language-runtime-compilation.md)。

如果您使用視覺效果C++編譯 ATL 應用程式，也可能會發生 D8045。 如需詳細資訊，請參閱[如何：遷移至/clr](../../dotnet/how-to-migrate-to-clr.md) 。
