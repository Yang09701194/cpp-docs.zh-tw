# <a name="mfc-com"></a>MFC COM

MFC 的子集旨在支援 COM,而大多數活動範本庫 (ATL) 專為 COM 編程而設計。 本部分主題介紹了 MFC 對 COM 的支援。

活動技術(如 ActiveX 控制件、活動文件包含、OLE 等)使用元件物件模型 (COM) 使軟體元件能夠在網路環境中相互交互,而不管它們使用何種語言。 活動技術可用於創建在桌面或 Internet 上執行的應用程式。 有關詳細資訊,請參閱[COM 或](../atl/introduction-to-com.md)[元件物件模型](/windows/win32/com/the-component-object-model)簡介。

活動技術包括客戶端和伺服器技術,包括:

- ActiveX 控制項是互動式物件,可用於容器(如網站)。 有關 ActiveX 控制件的詳細資訊,請參閱:

  - [MFC ActiveX 控制項](../mfc/mfc-activex-controls.md)

  - [網際網路上的 ActiveX 控制項](../mfc/activex-controls-on-the-internet.md)

  - [概述:互聯網](../mfc/mfc-internet-programming-basics.md)

  - [升級要在網際網路上使用的現有 ActiveX 控制項](../mfc/upgrading-an-existing-activex-control.md)

  - [除錯 ActiveX 控制項](/visualstudio/debugger/how-to-debug-an-activex-control)

- 活動文本控制瀏覽器或伺服器中一個或多個 ActiveX 控制件的集成行為。 有關活動文稿的詳細資訊,請參閱 Internet[上的活動技術](../mfc/active-technology-on-the-internet.md)。

- [自動化](../mfc/automation.md)(以前稱為 OLE 自動化)使一個應用程式能夠操作在另一個應用程式中實現的物件,或者"公開"物件,以便可以對其進行操作。

   自動物件可能是本地的或遠端的(在通過網路可訪問的另一台電腦上)。 Aurotmation 可供 OLE 和 COM 物件使用。

- 本節還提供有關如何使用 MFC 寫入 COM 元件的資訊,例如,在[連接點](../mfc/connection-points.md)中。

有關仍稱為 OLE 與現在稱為活動技術的討論,請參閱[OLE](../mfc/ole-in-mfc.md)上的主題。
