## <a name="customization-tips"></a>自訂秘訣

以下自訂提示與 繼承`CBasePane Class`的與任何類別有關:

- 建立窗格時,可以應用幾種新樣式:

  - AFX_CBRS_FLOAT使窗格浮動。

  - AFX_CBRS_AUTOHIDE啟用自動隱藏模式。

  - AFX_CBRS_CLOSE使窗格能夠關閉(隱藏)。

  這些是可以與位-OR 操作組合的標誌。

`CBasePane`實現以下虛擬布爾方法以反映這些標誌[:CBasePane::可以關閉](#canbeclosed)[,CBasePane::可以自動隱藏](#canautohide)[,CBasePane::可以浮動](#canfloat)。 您可以在派生類中重寫它們以自定義其行為。

- 您可以通過重寫[CBasePane::canAcceptPane](#canacceptpane)來自定義停靠行為。 讓窗格從此方法返回 FALSE,以防止另一個窗格停靠到它。

- 如果要創建無法浮動的靜態窗格,並且阻止任何其他窗格在它之前停靠(類似於 OutlookDemo 示例中的 Outlook 欄),請將其創建為非浮動,並重寫[CBasePane::DoesAllowDynInsert 之前](#doesallowdyninsertbefore)返回 FALSE。 如果創建窗格時沒有AFX_CBRS_FLOAT樣式,則默認實現將返回 FALSE。

- 使用 -1 以外的指示創建所有窗格。

- 要確定窗格可見性,請使用[CBasePane::是可見的](#isvisible)。 它正確處理選項卡式和自動隱藏模式下的可見性狀態。

- 如果要建立非浮動可調整大小的窗格,請建立它,而不AFX_CBRS_FLOAT樣式,並呼叫[CFrameWnd::DockControlBar](../../mfc/reference/cframewnd-class.md#dockcontrolbar)。

- 要從停靠佈局中排除窗格或從停靠欄中刪除工具列,請呼叫[CBasePane:::undockPane](#undockpane)。 不要為自動隱藏模式下的窗格或駐留在選項卡式視窗選項卡中的窗格調用此方法。

- 如果要浮動或取消停靠處於自動隱藏模式的窗格,則必須調用[CDockablePane::在](../../mfc/reference/cdockablepane-class.md#setautohidemode)調用[CBasePane::FloatPane](#floatpane)或[CBasePane:undockPane](#undockpane)之前,將 FALSE 設置為第一個參數。
