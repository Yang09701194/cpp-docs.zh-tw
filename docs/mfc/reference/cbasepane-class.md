## <a name="remarks"></a>備註

如果您想要建立支援 MFC 中可用擴充停駐功能的窗格類別, 則必須從`CBasePane` [CPane 類別](../../mfc/reference/cpane-class.md)衍生它。

## <a name="customization-tips"></a>自訂秘訣

下列自訂提示是與繼承`CBasePane Class`自它的任何類別相關的:

- 當您建立窗格時, 您可以套用數個新的樣式:

  - AFX_CBRS_FLOAT 會使窗格成為浮動。

  - AFX_CBRS_AUTOHIDE 啟用自動隱藏模式。

  - AFX_CBRS_CLOSE 可讓窗格關閉 (隱藏)。

  這些是您可以與位 OR 運算結合的旗標。

`CBasePane`會執行下列虛擬布林值方法來反映這些旗標:[CBasePane:: CanBeClosed](#canbeclosed)、 [CBasePane:: CanAutoHide](#canautohide)、 [CBasePane:: CanFloat](#canfloat)。 您可以在衍生類別中覆寫它們, 以自訂其行為。

- 您可以藉由覆寫[CBasePane:: CanAcceptPane](#canacceptpane)來自訂停駐行為。 讓您的窗格從這個方法傳回 FALSE, 以避免另一個窗格停駐于其中。

- 如果您想要建立無法浮動的靜態窗格, 並防止任何其他窗格在它之前停駐 (類似于 OutlookDemo 範例中的 Outlook 橫條), 請將它建立為非浮動並覆寫[CBasePane::D oesallowdyninsertbefore](#doesallowdyninsertbefore)以傳回FALSE. 如果建立不含 AFX_CBRS_FLOAT 樣式的窗格, 則預設的執行會傳回 FALSE。

- 建立識別碼不是-1 的所有窗格。

- 若要判斷窗格可見度, 請使用[CBasePane:: IsVisible](#isvisible)。 它會正確處理索引標籤式和自動隱藏模式中的可見度狀態。

- 如果您想要建立非浮動的可調整大小窗格中，建立不含 AFX_CBRS_FLOAT 樣式和呼叫[CFrameWnd::DockControlBar](../../mfc/reference/cframewnd-class.md#dockcontrolbar)。

- 若要從停駐版面配置中排除窗格, 或從其銜接列移除工具列, 請呼叫[CBasePane:: UndockPane](#undockpane)。 請勿針對自動隱藏模式中的窗格, 或位於索引標籤式視窗索引標籤中的窗格, 呼叫這個方法。

- 如果您想要浮動或取消停駐處於自動隱藏模式的窗格, 您必須在呼叫[CBasePane:: FloatPane](#floatpane)或[CBasePane:: UndockPane](#undockpane)之前, 以 FALSE 做為第一個引數來呼叫[CDockablePane:: SetAutoHideMode](../../mfc/reference/cdockablepane-class.md#setautohidemode) 。
