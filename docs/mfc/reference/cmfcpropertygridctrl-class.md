# <a name="cmfcpropertygridctrl-class"></a>CMFCPropertyGridCtrl 類別
[!INCLUDE[cpp_fp_under_construction](../../mfc/reference/includes/cpp_fp_under_construction_md.md)]  
  
 支援可依字母順序或階層順序顯示屬性的可編輯屬性方格控制項。  
  
## <a name="syntax"></a>語法  
  
```  
class CMFCPropertyGridCtrl : public CWnd  
```

## <a name="remarks"></a>備註  
 `CMFCPropertyGridCtrl`類別會顯示屬性方格控制項，其中包含可編輯的屬性，衍生自[CMFCPropertyGridProperty](../../mfc/reference/cmfcpropertygridproperty-class.md)類別。 每個屬性可以表示的類型，它可以包含子項目。 屬性方格控制項支援的可調整大小區域底部，可顯示所選屬性的描述。  
  
 若要使用屬性方格控制項，建構`CMFCPropertyGridCtrl`物件，然後呼叫[CMFCPropertyGridCtrl::Create](#create)方法。 使用[CMFCPropertyGridCtrl::AddProperty](#addproperty)方法，將屬性加入至清單。  
  
## <a name="selection-properties"></a>選取項目屬性  
 屬性項目而不是代表值，就可以開始對話方塊，讓使用者選取色彩、 檔案或字型。  
  
 下表列出四個選取項目屬性的型別︰  
  
|類別|說明|  
|-----------|-----------------|  
|[CMFCPropertyGridProperty 類別](../../mfc/reference/cmfcpropertygridproperty-class.md)|一般用途的屬性，用於指定的字串、 布林值，日期值，依此類推。|  
|[CMFCPropertyGridColorProperty 類別](../../mfc/reference/cmfcpropertygridcolorproperty-class.md)|此屬性，用來選取色彩值。|  
|[CMFCPropertyGridFileProperty 類別](../../mfc/reference/cmfcpropertygridfileproperty-class.md)|此屬性，用來選取檔案。|  
|[CMFCPropertyGridFontProperty 類別](../../mfc/reference/cmfcpropertygridfontproperty-class.md)|使用選取的字型屬性。|

##  <a name="addproperty"></a>CMFCPropertyGridCtrl::AddProperty  
 將新屬性加入至屬性方格控制項。  
  
```  
int AddProperty(
    CMFCPropertyGridProperty* pProp,  
    BOOL bRedraw=TRUE,  
    BOOL bAdjustLayout=TRUE);
```  
  
### <a name="parameters"></a>參數  
 [in] `pProp`  
 屬性的指標。  
  
 [in] `bRedraw`  
 `TRUE`若要立即; 重繪屬性否則， `FALSE`。 預設值是 `TRUE`。  
  
 [in] `bAdjustLayout`  
 `TRUE`若要重新計算如何繪製文字和屬性的值，然後畫出屬性。`FALSE`繪製屬性使用現有的計算。 預設值是 `TRUE`。

 如果此方法成功，這個屬性加入其中，在屬性方格控制項位置的以零起始的索引反之則為-1。  
  
### <a name="remarks"></a>備註  
 這個方法會加入至指定的屬性，屬性方格控制項中的屬性清單的結尾的指標。 請勿摧毀屬性或允許它們方格控制項被終結之前超出範圍。 當您完成屬性方格控制項時，呼叫[CMFCPropertyGridCtrl::RemoveAll](#removeall)刪除所有加入的屬性。 如果指定的屬性已加入至清單，AddProperty 方法將會失敗。

##  <a name="hittest"></a>CMFCPropertyGridCtrl::HitTest  
 擷取對應到屬性方格控制項項目，如果指定的點是項目中的屬性物件的指標。 這個方法也會指出包含點的屬性方格控制項中的區域。  
  
```  
CMFCPropertyGridProperty* HitTest(
    CPoint pt,  
    CMFCPropertyGridProperty::ClickArea* pnArea=NULL,  
    BOOL bPropsOnly=FALSE) const;  
```  
  
### <a name="parameters"></a>參數  
 [in] `pt`  
 用戶端座標中的點。  
  
 [in、out] `pnArea`  
 指標`ClickArea`變數。 當這個方法傳回時，變數會指出*屬性區域*，其中包含指定的點。 如需屬性區域的詳細資訊，請參閱 < 備註 >。  
  
 [in] `bPropsOnly`  
 `TRUE`若要測試只屬性的區域。`FALSE`測試*描述區域*如果指定的點不是在 [屬性] 區域。 預設值是 `FALSE`。 如需 [描述] 區域的詳細資訊，請參閱 < 備註 >。  
  
### <a name="return-value"></a>傳回值  
 如果`bPropsOnly`參數是`TRUE`和指定的點位於屬性區域中，傳回的值是對應的屬性物件的指標。 此外，`pnArea`參數設定為包含指定的點的特定區域。 否則，傳回的值是`NULL`和`pnArea`不會修改參數。  
  
 如果`bPropsOnly`參數是`FALSE`，傳回值永遠是`NULL`。 不過，如果指定的點是在 [描述] 區域中，`pnArea`參數設為`CMFCPropertyGridProperty::ClickDescription`。  
  
### <a name="remarks"></a>備註  
 這個詞彙*屬性區域*指的是其中一個名稱值，或展開方塊部分的屬性方格控制項項目。 *描述區域*是在屬性方格控制項底部區域。 當您按一下屬性方格控制項項目時，[描述] 區域會顯示對應屬性的描述。  
  
 這個方法會設定變數的值，`pnArea`參數所指向。 下表列出可能的值和對應的區域。  
  
|值|區域|  
|-----------|----------|  
|`ClickArea::ClickExpandBox`|屬性會展開方塊控制項。|  
|`ClickArea::ClickName`|屬性名稱|  
|`ClickArea::ClickValue`|屬性值。|  
|`CMFCPropertyGridProperty::ClickDescription`|屬性方格控制項描述區域。|

 指定是否在目前的屬性方格控制項中顯示一組屬性的類別名稱的完整寬度。  
  
```  
void SetGroupNameFullWidth(
    BOOL bGroupNameFullWidth = TRUE,  
    BOOL bRedraw = TRUE);
```  
  
### <a name="parameters"></a>參數  
 [in] `bGroupNameFullWidth`  
 `TRUE`若要顯示完整的寬度，不論屬性名稱資料行的寬度，類別目錄名稱。 `FALSE`若要限制類別名稱的屬性名稱資料行寬度的寬度。 預設值是 `TRUE`。  
  
 [in] `bRedraw`  
 `TRUE`若要立即; 更新屬性方格控制項`FALSE`來更新控制下, 一個重繪事件時就會發生。 預設值是 `TRUE`。  
  
### <a name="remarks"></a>備註  
 屬性方格控制項包含可調整大小*屬性名稱*資料行和*屬性值*資料行。 名稱資料行的結尾也是值資料行的開頭。 若要調整大小之資料行，拖曳資料行之間的框線。  
  
 條款*群組名稱*和*類別名稱*會交替使用這個方法。 類別名稱會顯示在朝九晚五一組相關的屬性和值的資料列。 這個方法會指定屬性名稱資料行的寬度是否也指定的寬度，顯示類別目錄名稱。
