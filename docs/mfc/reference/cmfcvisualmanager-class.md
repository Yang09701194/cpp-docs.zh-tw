# <a name="cmfcvisualmanager-class"></a>CMFCVisualManager 類別
提供在全域層級上變更應用程式外觀的支援。 `CMFCVisualManager` 類別會使用一致的樣式，與提供繪製應用程式 GUI 控制項指令的類別搭配使用。 這些其他類別稱為視覺管理員，並且繼承自 `CMFCBaseVisualManager`。  

 [!INCLUDE[cpp_fp_under_construction](../../mfc/reference/includes/cpp_fp_under_construction_md.md)]  
  
## <a name="syntax"></a>語法  
  
```  
class CMFCVisualManager : public CMFCBaseVisualManager  
```

## <a name="remarks"></a>備註  
 因為`CMFCVisualManager`類別控制應用程式的 GUI，每個應用程式可以擁有的任一個一個執行個體`CMFCVisualManager`，或從衍生類別的一個執行個體`CMFCVisualManager`。 您的應用程式也可以運作而不`CMFCVisualManager`。 使用靜態方法`GetInstance`來取得目前的指標`CMFCVisualManager`-衍生物件。  
  
 若要變更您的應用程式的外觀，您必須使用其他提供繪製應用程式的視覺化元素的所有方法的類別。 這些類別的一些範例包括[CMFCVisualManagerOfficeXP 類別](../../mfc/reference/cmfcvisualmanagerofficexp-class.md)， [CMFCVisualManagerOffice2003 類別](../../mfc/reference/cmfcvisualmanageroffice2003-class.md)，和[CMFCVisualManagerOffice2007 類別](../../mfc/reference/cmfcvisualmanageroffice2007-class.md)。 當您想要變更您的應用程式的外觀時，其中一個這些視覺管理員傳遞至方法`SetDefaultManager`。 如需示範如何您的應用程式可以模擬 Microsoft Office 2003 的外觀的範例，請參閱[CMFCVisualManagerOffice2003 類別](../../mfc/reference/cmfcvisualmanageroffice2003-class.md)。  
  
 所有的繪圖方法都是虛擬的。 這可讓您建立自訂的視覺化樣式的應用程式的 GUI。 如果您想要建立您自己的視覺化樣式，衍生的其中一個視覺管理員類別的類別並覆寫您想要變更繪圖方法。

CMFCVisualManager::GetInstance ()->RedrawAll ();
