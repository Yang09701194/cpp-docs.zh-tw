# <a name="styles-used-by-mfc"></a>MFC 使用的樣式

在創建相應的 MFC 物件時,請使用以下樣式標誌指定視窗或控制元件外觀和行為。 在大多數情況下,這些樣式設置在類`Create`函數的*dwStyle*參數中。

## <a name="button-styles"></a><a name="button-styles"></a>按鈕樣式

按鈕樣式適用於[CButton 類](../../mfc/reference/cbutton-class.md)物件,如單選按鈕、複選框和按鈕。 在[CButton:::建立](../../mfc/reference/cbutton-class.md#create)的*dwStyle*參數中指定樣式的組合。 有關 Windows 中的按鈕樣式的詳細資訊,請參閱[按鍵樣式 (Windows)](/windows/win32/Controls/button-styles)。

### <a name="button-types"></a>按鈕類型

下表列出了按鈕類型。 您可以選擇以下選項之一。 如果不指定按鈕類型,則默認值為BS_PUSHBUTTON。

|類型|描述|
|----------|-----------------|
|BS_3STATE|創建一個複選框按鈕,該按鈕具有三種狀態:BST_CHECKED、BST_INDETERMINATE和BST_UNCHECKED。 按下該按鈕會向所有者視窗發送BN_CLICKED通知,但不會更改按鈕的狀態。 預設情況下,關聯的文本顯示在複選框的右側。 要將文本顯示在複選框的左側,請使用BS_LEFTTEXT或BS_RIGHTBUTTON樣式。|
|BS_AUTO3STATE|創建一個複選框按鈕,該按鈕具有三種狀態:BST_CHECKED、BST_INDETERMINATE和BST_UNCHECKED。 按下這個按鈕會向所有者視窗發送BN_CLICKED通知並更改按鈕的狀態。 該按鈕按BST_CHECKED、BST_INDETERMINATE和BST_UNCHECKED的順序表示迴圈。 預設情況下,關聯的文本顯示在複選框的右側。 要將文本顯示在複選框的左側,請使用BS_LEFTTEXT或BS_RIGHTBUTTON樣式。|
|BS_AUTOCHECKBOX|創建一個複選框按鈕,該按鈕具有兩種狀態:BST_CHECKED和BST_UNCHECKED。 按下這個按鈕會向所有者視窗發送BN_CLICKED通知並更改按鈕的狀態。 預設情況下,關聯的文本顯示在複選框的右側。 要將文本顯示在複選框的左側,請使用BS_LEFTTEXT或BS_RIGHTBUTTON樣式。|
|BS_AUTORADIOBUTTON|創建具有兩種狀態的單選按鈕:BST_CHECKED和BST_UNCHECKED。 單選按鈕通常以組形式使用,每個組一次最多有一個選中的選項。 按下這個按鈕會向所有者視窗發送BN_CLICKED通知,將單擊的單選按鈕的狀態設置為BST_CHECKED,並將按鈕組中所有其他單選按鈕的狀態設置為BST_UNCHECKED。 默認情況下,關聯的文本顯示在單選按鈕的右側。 要在單選按鈕的左側顯示文本,請使用BS_LEFTTEXT或BS_RIGHTBUTTON樣式。|
|BS_CHECKBOX|創建一個複選框按鈕,該按鈕具有兩種狀態:BST_CHECKED和BST_UNCHECKED。 按下該按鈕會向所有者視窗發送BN_CLICKED通知,但不會更改按鈕的狀態。 預設情況下,關聯的文本顯示在複選框的右側。 要將文本顯示在複選框的左側,請使用BS_LEFTTEXT或BS_RIGHTBUTTON樣式。|
|BS_COMMANDLINK|建立命令連結按鈕。 命令連結按鈕是特定於 Windows Vista 的命令按鈕,在主文本的左側顯示一個綠色箭頭,並在主文本下方顯示註釋。 您可以使用[CButton::setNote](../../mfc/reference/cbutton-class.md#setnote)設定註解文本。|
|BS_DEFCOMMANDLINK|建立命令連結按鈕。 命令連結按鈕是特定於 Windows Vista 的命令按鈕,在主文本的左側顯示一個綠色箭頭,並在主文本下方顯示註釋。 您可以使用[CButton::setNote](../../mfc/reference/cbutton-class.md#setnote)設定註解文本。 如果按鈕在對話框中,則按下 ENTER 鍵會向對話框發送BN_CLICKED通知,即使該按鈕沒有輸入焦點也是如此。|
|BS_DEFPUSHBUTTON|創建具有粗黑色邊框的命令按鈕。 如果按鈕在對話框中,則按下 ENTER 鍵會向對話框發送BN_CLICKED通知,即使該按鈕沒有輸入焦點也是如此。|
|BS_DEFSPLITBUTTON|創建拆分按鈕。 拆分按鈕是特定於 Windows Vista 的命令按鈕,其中包含與下拉箭頭相鄰的按鈕。 按下該按鈕時,將執行預設命令。 按下拉箭頭時,將顯示其他命令的功能表。 如果拆分按鈕位於對話框中,則按下 ENTER 鍵也會向對話框傳送BN_CLICKED通知,即使該按鈕沒有輸入焦點|
|BS_GROUPBOX|創建一個矩形,其中可以對其他按鈕進行分組。 與此樣式關聯的文本顯示在矩形的左上角。|
|BS_OWNERDRAW|建立擁有者繪製的按鈕。 當按鈕的可視`DrawItem`方面已更改時,框架將調用方法。 使用`CBitmapButton`類時必須設置此樣式。|
|BS_PUSHBUTTON|創建一個命令按鈕,在使用者按一下該按鈕時向所有者視窗發送BN_CLICKED通知。|
|BS_RADIOBUTTON|創建具有兩種狀態的單選按鈕:BST_CHECKED和BST_UNCHECKED。 單選按鈕通常以組形式使用,每個組一次最多有一個選中的選項。 按下這個按鈕會向所有者視窗發送BN_CLICKED通知,但不會自動更改組中任何按鈕的狀態。 默認情況下,關聯的文本顯示在單選按鈕的右側。 要在單選按鈕的左側顯示文本,請使用BS_LEFTTEXT或BS_RIGHTBUTTON樣式。|
|BS_SPLITBUTTON|創建拆分按鈕。 拆分按鈕是特定於 Windows Vista 的命令按鈕,其中包含與下拉箭頭相鄰的按鈕。 按下該按鈕時,將執行預設命令。 按下拉箭頭時,將顯示其他命令的功能表。|
|BS_USERBUTTON|已過時,但提供了與 16 位版本的 Windows 的相容性。 基於 Win32 的應用程式應該使用BS_OWNERDRAW。|

### <a name="radio-button-and-check-box-styles"></a>點選按鈕與選擇方塊樣式

下表列出了特定於單選按鈕和複選框的樣式。 在所有其他按鈕類型中,這些樣式將被忽略。 您可以選擇以下一種或多項。

|Style|描述|
|-----------|-----------------|
|BS_LEFTTEXT|與單選按鈕或複選框樣式結合使用時,文本將顯示在單選按鈕或複選框的左側。|
|BS_RIGHTBUTTON|與單選按鈕或複選框樣式結合使用時,文本將顯示在單選按鈕或複選框的左側。 此樣式與BS_LEFTTEXT樣式相同。|
|BS_PUSHLIKE|使複選框或單選按鈕的外觀和操作類似於命令按鈕。 當該按鈕的狀態BST_CHECKED時,該按鈕在BST_INDETERMINATE狀態時按下並變暗,並在其狀態BST_UNCHECKED時釋放。|

### <a name="button-text-alignment-styles"></a>按鈕文字對齊樣式

下表列出了水平和垂直文本對齊選項。 您可以選擇以下選項之一。

|Style|描述|
|-----------|-----------------|
|BS_LEFT|左側對齊按鈕矩形中的文本。 但是,如果按鈕是一個複選框或單選按鈕,沒有BS_RIGHTBUTTON樣式,則文本在複選框或單選按鈕的右側保持對齊。|
|BS_RIGHT|右對齊按鈕矩形中的文字。 但是,如果按鈕是一個複選框或單選按鈕,沒有BS_RIGHTBUTTON樣式,則文本在複選框或單選按鈕的右側右對齊。|
|BS_CENTER|在按鈕矩形中水準居中文字。|
|BS_TOP|將文字放在按鈕矩形的頂部。|
|BS_BOTTOM|將文字放在按鈕矩形的底部。|
|BS_VCENTER|在按鈕矩形中垂直居中文字。|

### <a name="button-content-options"></a>按鈕內容選項

下表列出了指示按鈕中顯示的選項。 僅顯示文本的按鈕類型將忽略這些樣式。 您可以選擇以下選項之一。

|Style|描述|
|-----------|-----------------|
|BS_BITMAP|指定按鈕顯示點陣圖。|
|BS_ICON|指定按鈕顯示圖示。|
|BS_TEXT|指定按鈕顯示文本。|

### <a name="other-button-options"></a>其他按鈕選項

下表列出了可用於任何按鈕類型的其他選項。 您可以選擇以下一種或多項。

|Style|描述|
|-----------|-----------------|
|BS_FLAT|指定按鈕是二維的,並且不會使用預設的圖帶繪製以創建三維圖像。|
|BS_MULTILINE|如果文字字串太長,無法位於按鈕矩形中的一行,則將按鈕文本換行到多行。|
|BS_NOTIFY|啟用按鈕,將BN_DBLCLK、BN_KILLFOCUS和BN_SETFOCUS通知消息發送到其父視窗。 請注意,無論是否指定此樣式,按鈕都會發送BN_CLICKED通知。|

## <a name="combo-box-styles"></a><a name="combo-box-styles"></a>組合盒樣式

MFC 中提供下列下拉式方塊樣式。 有關 Windows 中的組合框樣式的詳細資訊,請參閱[組合框樣式 (Windows)。](/windows/win32/Controls/combo-box-styles)

|Style|描述|
|-----------|-----------------|
|CBS_AUTOHSCROLL|當使用者在行尾鍵入字元時,自動將編輯控制件中的文本向右滾動。 如果未設定此樣式，則只會允許符合矩形界限的文字。|
|CBS_DISABLENOSCROLL|當清單框不包含足夠的專案以滾動時,清單框將顯示禁用的垂直滾動條。 若沒有這個樣式，在清單方塊中未包含足夠的項目時，捲軸會隱藏。|
|CBS_DROPDOWN|與CBS_SIMPLE類似,除非用戶選擇編輯控件旁邊的圖示,否則不會顯示列表框。|
|CBS_DROPDOWNLIST|與CBS_DROPDOWN類似,只不過編輯控制項替換為在列表框中顯示當前選擇的靜態文本項。|
|CBS_HASSTRINGS|擁有者繪製組合框包含由字串組成的項。 下拉式方塊會維護記憶體和字串的指標，讓應用程式可以使用 `GetText` 成員函式來擷取特定項目的文字。|
|CBS_LOWERCASE|轉換為小寫選擇欄位和清單中的所有文本。|
|CBS_NOINTEGRALHEIGHT|指定組合框的大小與應用程式創建組合框時指定的大小完全相同。 一般而言，Windows 會調整下拉式方塊的大小，使下拉式方塊不會只顯示一部分的項目。|
|CBS_OEMCONVERT|在組合框編輯控制項中輸入的文本將從 ANSI 字元集轉換為 OEM 字元集,然後轉換回 ANSI。 如此可確保當應用程式呼叫 `AnsiToOem` Windows 函式將下拉式方塊中的 ANSI 字串轉換成 OEM 字元時，會進行適當的字元轉換。 此樣式是最適合用於包含檔案名稱的下拉式方塊，而且只適用於使用 CBS_SIMPLE 或 CBS_DROPDOWN 樣式建立的下拉式方塊。|
|CBS_OWNERDRAWFIXED|清單框的所有者負責繪製其內容;清單框中的專案都具有相同的高度。|
|CBS_OWNERDRAWVARIABLE|清單框的所有者負責繪製其內容;清單框中的專案高度是可變的。|
|CBS_SIMPLE|清單框隨時顯示。 編輯控制項會顯示清單方塊中目前的選取範圍。|
|CBS_SORT|自動對輸入到清單框中的字串進行排序。|
|CBS_UPPERCASE|轉換為大寫,選擇欄位和清單中的所有文本。|

## <a name="edit-styles"></a><a name="edit-styles"></a>編輯樣式

|SS_NOPREFIX|除非指定此樣式,否則 Windows 會將控制項文本中的任何放大字元(&)解釋為快捷鍵前綴字元。 在這種情況下,將刪除放大器,並下劃下劃線中的下一個字元。 如果靜態控件要包含不需要此功能的文本,則可以添加SS_NOPREFIX。 此靜態控制樣式可以包含在任何定義的靜態控制項中。 通過使用位或運算符,可以將SS_NOPREFIX與其他樣式組合在一起。 當檔名或其他可能包含放大器且需要在對話框中的靜態控件中顯示時,最常使用此方法。|
|SS_NOTIFY|當用戶按下或雙擊控制項時,將父視窗STN_CLICKED、STN_DBLCLK、STN_DISABLE和STN_ENABLE通知消息發送。|
|SS_OWNERDRAW|指定靜態控制件的所有者負責繪製控制件。 每當需要繪製控件時,所有者視窗都會收到WM_DRAWITEM消息。|
|SS_REALSIZEIMAGE|防止靜態圖示或位圖控件(即具有SS_ICON或SS_BITMAP樣式的靜態控件)在載入或繪製時調整大小。 如果圖示或位圖大於目標區域,則剪切圖像。|
|SS_RIGHT|指定簡單的矩形,並在矩形中向右顯示給定的文本。 文字在顯示之前已設置格式。 延伸到行尾的單詞將自動換行到下一條沖右線的開頭。|
|SS_RIGHTJUST|指定具有SS_BITMAP或SS_ICON樣式的靜態控件的右下角在調整控制項大小時保持固定。 只有頂部和左側進行調整以適應新的位圖或圖示。|
|SS_SIMPLE|指定簡單的矩形,並在矩形中顯示一列左衝線文字。 文本行不能以任何方式縮短或更改。 (控制件的父視窗或對話方塊不得處理WM_CTLCOLOR消息。|
|SS_SUNKEN|圍繞靜態控件繪製半凹陷邊框。|
|SS_USERITEM|指定使用者定義的項。|
|SS_WHITEFRAME|指定繪製框架的框與視窗背景的顏色相同。 預設為白色。|
|SS_WHITERECT|指定矩形,該矩形填充用於填充視窗背景的顏色。 預設為白色。|
|SS_WORDELLIPSIS|截緊不適合的文本並添加省略號。|

## <a name="window-styles"></a><a name="window-styles"></a>視窗樣式

視窗樣式應用於[CWnd 類](../../mfc/reference/cwnd-class.md)物件。 在[CWnd 的](../../mfc/reference/cwnd-class.md#create) *dwStyle*參數中指定樣式的群組:建立或[CWnd:createEx](../../mfc/reference/cwnd-class.md#createex)。 有關 Windows 中視窗樣式的詳細資訊,請參閱[視窗樣式 (Windows)](/windows/win32/winmsg/window-styles)。

|Style|描述|
|-----------|-----------------|
|WS_BORDER|創建具有邊框的視窗。|
|WS_CAPTION|創建具有標題列的視窗(表示WS_BORDER樣式)。 不能與WS_DLGFRAME樣式一起使用。|
|WS_CHILD|創建子視窗。 不能與WS_POPUP樣式一起使用。|
|WS_CHILDWINDOW|與WS_CHILD風格相同。|
|WS_CLIPCHILDREN|在父視窗內繪圖時，應排除子視窗佔用的區域。 創建父視窗時使用。|
|WS_CLIPSIBLINGS|剪輯子視窗彼此;也就是說,當特定子視窗收到繪製消息時,WS_CLIPSIBLINGS樣式將所有其他重疊的子視窗從要更新的子視窗區域中夾出。 (如果未給出WS_CLIPSIBLINGS,並且子視窗重疊,則當您在子視窗的工作區中繪製時,可以在相鄰子視窗的工作區中繪製。僅適用於WS_CHILD樣式。|
|WS_DISABLED|建立一個一開始即停用的視窗。|
|WS_DLGFRAME|創建具有雙邊框但沒有標題的視窗。|
|WS_GROUP|指定一組控制項的第一個控制項,用戶可以在其中使用箭頭鍵從一個控制項移動到下一個控制項。 在第一個控制項屬於同一組後,使用WS_GROUP樣式 FALSE 定義的所有控制項。 具有WS_GROUP樣式的下一個控制項將啟動下一個組(即,一個組結束下一個組開始的位置)。|
|WS_HSCROLL|創建具有水平滾動條的視窗。|
|WS_ICONIC|創建最初最小化的視窗。 與WS_MINIMIZE風格相同。|
|WS_MAXIMIZE|創建最大大小的視窗。|
|WS_MAXIMIZEBOX|創建具有"最大化"按鈕的視窗。|
|WS_MINIMIZE|創建最初最小化的視窗。 僅適用於WS_OVERLAPPED樣式。|
|WS_MINIMIZEBOX|建立具有「最小化」按鈕的視窗。|
|WS_OVERLAPPED|創建重疊視窗。 重疊的視窗通常具有標題和邊框。|
|WS_OVERLAPPEDWINDOW|創建具有WS_OVERLAPPED、WS_CAPTION、WS_SYSMENU、WS_THICKFRAME、WS_MINIMIZEBOX和WS_MAXIMIZEBOX樣式的重疊視窗。|
|WS_POPUP|創建彈出視窗。 不能與WS_CHILD樣式一起使用。|
|WS_POPUPWINDOW|創建具有WS_BORDER、WS_POPUP和WS_SYSMENU樣式的彈出視窗。 WS_CAPTION樣式必須與WS_POPUPWINDOW樣式相結合,以使"控制件"功能表可見。|
|WS_SIZEBOX|建立具有大小調整邊框的視窗。 與WS_THICKFRAME風格相同。|
|WS_SYSMENU|建立標題列中包含"控制"選單框的視窗。 僅用於帶標題列的視窗。|
|WS_TABSTOP|指定使用者可以使用 TAB 鍵移動的任意數量的控制項之一。 TAB 鍵將使用者移動到WS_TABSTOP樣式指定的下一個控制項。|
|WS_THICKFRAME|建立具有粗框架的視窗,可用於調整視窗的大小。|
|WS_TILED|創建重疊視窗。 重疊的視窗具有標題列和邊框。 與WS_OVERLAPPED風格相同。|
|WS_TILEDWINDOW|創建具有WS_OVERLAPPED、WS_CAPTION、WS_SYSMENU、WS_THICKFRAME、WS_MINIMIZEBOX和WS_MAXIMIZEBOX樣式的重疊視窗。 與WS_OVERLAPPEDWINDOW風格相同。|
|WS_VISIBLE|建立一個一開始即可見的視窗。|
|WS_VSCROLL|創建具有垂直滾動條的視窗。|

## <a name="extended-window-styles"></a><a name="extended-window-styles"></a>延伸視窗樣式

擴展視窗樣式適用於[CWnd 類](../../mfc/reference/cwnd-class.md)物件。 在 CWnd 的*dwExStyle*參數中指定樣式的群組[::createEx](../../mfc/reference/cwnd-class.md#createex)。 有關 Windows 中擴展視窗樣式的詳細資訊,請參閱[擴展視窗樣式 (Windows)。](/windows/win32/winmsg/extended-window-styles)

|Style|描述|
|-----------|-----------------|
|WS_EX_ACCEPTFILES|指定使用此樣式建立的視窗接受拖放檔。|
|WS_EX_APPWINDOW|當視窗可見時,將頂層窗口強制強制到任務列上。|
|WS_EX_CLIENTEDGE|指定視窗具有 3D 外觀 ,即具有凹陷邊緣的邊框。|
|WS_EX_CONTEXTHELP|在視窗的標題列中包含一個問號。 當用戶單擊問號時,游標將變為帶有指標的問號。 如果使用者按一下子視窗，子視窗會收到 WM_HELP 訊息。|
|WS_EX_CONTROLPARENT|允許使用者使用 TAB 鍵在視窗的子窗口之間導航。|
|WS_EX_DLGMODALFRAME|指定具有雙邊框的視窗,當您在*dwStyle*參數中指定WS_CAPTION樣式標誌時,可以使用標題列創建該視窗(可選)。|
|WS_EX_LAYERED|視窗是[分層視窗](/windows/win32/winmsg/window-features)。 如果視窗的[類樣式](/windows/win32/winmsg/about-window-classes)為CS_OWNDC或CS_CLASSDC,則無法使用此樣式。 但是,Microsoft Windows 8 確實支援子視窗的WS_EX_LAYERED樣式,其中以前的 Windows 版本僅支援頂級視窗。|
|WS_EX_LEFT|提供視窗泛型左對齊屬性。 這是預設值。|
|WS_EX_LEFTSCROLLBAR|將垂直滾動條放在工作區的左側。|
|WS_EX_LTRREADING|使用從左至右的閱讀順序屬性顯示視窗文本。 這是預設值。|
|WS_EX_MDICHILD|創建 MDI 子視窗。|
|WS_EX_NOPARENTNOTIFY|指定使用此樣式創建的子視窗在創建或銷毀子視窗時不會將WM_PARENTNOTIFY訊息發送到其父視窗。|
|WS_EX_OVERLAPPEDWINDOW|結合WS_EX_CLIENTEDGE和WS_EX_WINDOWEDGE風格|
|WS_EX_PALETTEWINDOW|結合了WS_EX_WINDOWEDGE和WS_EX_TOPMOST風格。|
|WS_EX_RIGHT|提供視窗泛型右對齊屬性。 這取決於視窗類別。|
|WS_EX_RIGHTSCROLLBAR|將垂直滾動條(如果存在)放在工作區的右側。 這是預設值。|
|WS_EX_RTLREADING|使用從右至左讀取順序屬性顯示視窗文本。|
|WS_EX_STATICEDGE|創建具有三維邊框樣式的視窗,用於不接受使用者輸入的項。|
|WS_EX_TOOLWINDOW|建立工具視窗,該視窗是用作浮動工具列的視窗。 工具視窗的標題列較一般標題列短，而且使用小字型繪製視窗標題。 工具視窗不會在工作列中顯示，當使用者按下 ALT+TAB 時也不會在出現的視窗中顯示。|
|WS_EX_TOPMOST|指定使用此樣式創建的視窗應放置在所有非最頂層視窗之上,並在關閉視窗時保持在視窗上方。 應用程式可以使用 `SetWindowPos` 成員函式來加入或移除此屬性。|
|WS_EX_TRANSPARENT|指定使用此樣式創建的視窗是透明的。 也就是說，在這個視窗下的任何視窗不會被遮蔽。 使用此樣式創建的視窗僅在更新其下的所有同級視窗後才會接收WM_PAINT消息。|
|WS_EX_WINDOWEDGE|指定視窗具有具有凸起邊的邊框。|
