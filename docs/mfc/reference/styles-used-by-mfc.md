# <a name="styles-used-by-mfc"></a>MFC 使用的樣式

當您建立對應的 MFC 物件時，請使用下列樣式旗標來指定視窗或控制面板和行為。 在大部分的情況下，這些樣式是在類別 `Create` 函數的*dwStyle*參數中設定。

## <a name="button-styles"></a>按鈕樣式

按鈕樣式適用于[CButton 類別](../../mfc/reference/cbutton-class.md)物件，例如選項按鈕、核取方塊和按鈕。 在[CButton：： Create](../../mfc/reference/cbutton-class.md#create)的*dwStyle*參數中指定樣式的組合。 如需 Windows 中按鈕樣式的詳細資訊，請參閱[按鈕樣式（Windows）](/windows/win32/Controls/button-styles)。

### <a name="button-types"></a>按鈕類型

下表列出按鈕類型。 您可以選擇性地選擇下列其中一項。 如果您未指定按鈕類型，預設值為 BS_PUSHBUTTON。

|類型|描述|
|----------|-----------------|
|BS_3STATE|建立具有三個狀態的核取方塊按鈕： BST_CHECKED、BST_INDETERMINATE 和 BST_UNCHECKED。 按一下按鈕，就會傳送 BN_CLICKED 通知給擁有者視窗，但不會變更按鈕的狀態。 根據預設，相關聯的文字會顯示在核取方塊的右邊。 若要在核取方塊的左邊顯示文字，請使用 BS_LEFTTEXT 或 BS_RIGHTBUTTON 樣式。|
|BS_AUTO3STATE|建立具有三個狀態的核取方塊按鈕： BST_CHECKED、BST_INDETERMINATE 和 BST_UNCHECKED。 按一下按鈕，就會將 BN_CLICKED 通知傳送至主控視窗，並變更按鈕的狀態。 按鈕狀態會以 BST_CHECKED、BST_INDETERMINATE 和 BST_UNCHECKED 的順序迴圈。 根據預設，相關聯的文字會顯示在核取方塊的右邊。 若要在核取方塊的左邊顯示文字，請使用 BS_LEFTTEXT 或 BS_RIGHTBUTTON 樣式。|
|BS_AUTOCHECKBOX|建立具有兩個狀態的核取方塊按鈕： BST_CHECKED 和 BST_UNCHECKED。 按一下按鈕，就會將 BN_CLICKED 通知傳送至主控視窗，並變更按鈕的狀態。 根據預設，相關聯的文字會顯示在核取方塊的右邊。 若要在核取方塊的左邊顯示文字，請使用 BS_LEFTTEXT 或 BS_RIGHTBUTTON 樣式。|
|BS_AUTORADIOBUTTON|建立具有兩個狀態的選項按鈕： BST_CHECKED 和 BST_UNCHECKED。 選項按鈕通常用於群組中，每個群組一次最多隻能有一個已核取的選項。 按一下按鈕會將 BN_CLICKED 通知傳送到主控視窗，將按下的選項按鈕的狀態設定為 [BST_CHECKED]，然後將按鈕群組中所有其他選項按鈕的狀態設定為 [BST_UNCHECKED]。 根據預設，相關聯的文字會顯示在選項按鈕的右邊。 若要在選項按鈕左邊顯示文字，請使用 BS_LEFTTEXT 或 BS_RIGHTBUTTON 樣式。|
|BS_CHECKBOX|建立具有兩個狀態的核取方塊按鈕： BST_CHECKED 和 BST_UNCHECKED。 按一下按鈕，就會傳送 BN_CLICKED 通知給擁有者視窗，但不會變更按鈕的狀態。 根據預設，相關聯的文字會顯示在核取方塊的右邊。 若要在核取方塊的左邊顯示文字，請使用 BS_LEFTTEXT 或 BS_RIGHTBUTTON 樣式。|
|BS_COMMANDLINK|建立命令連結按鈕。 命令連結按鈕是 Windows Vista 特定的命令按鈕，會在主要文字左邊顯示綠色箭號，並在主要文字底下提供附注。 您可以使用[CButton：： SetNote](../../mfc/reference/cbutton-class.md#setnote)來設定附注文字。|
|BS_DEFCOMMANDLINK|建立命令連結按鈕。 命令連結按鈕是 Windows Vista 特定的命令按鈕，會在主要文字左邊顯示綠色箭號，並在主要文字底下提供附注。 您可以使用[CButton：： SetNote](../../mfc/reference/cbutton-class.md#setnote)來設定附注文字。 如果按鈕在對話方塊中，按 ENTER 鍵會將 BN_CLICKED 通知傳送至對話方塊，即使按鈕沒有輸入焦點也一樣。|
|BS_DEFPUSHBUTTON|建立具有繁重黑色框線的命令按鈕。 如果按鈕在對話方塊中，按 ENTER 鍵會將 BN_CLICKED 通知傳送至對話方塊，即使按鈕沒有輸入焦點也一樣。|
|BS_DEFSPLITBUTTON|建立分割按鈕。 分割按鈕是 Windows Vista 特定的命令按鈕，其中包含下拉式箭號旁的按鈕。 當您按一下按鈕時，就會執行預設命令。 當您按一下下拉箭號時，就會出現其他命令的功能表。 如果 [分割] 按鈕在對話方塊中，按 ENTER 鍵會將 BN_CLICKED 通知傳送至對話方塊，即使按鈕沒有輸入焦點也一樣|
|BS_GROUPBOX|建立可將其他按鈕分組的矩形。 與這個樣式相關聯的文字會顯示在矩形的左上角。|
|BS_OWNERDRAW|建立主控描繪按鈕。 當按鈕的視覺外觀變更時，架構會呼叫 `DrawItem` 方法。 當您使用 `CBitmapButton` 類別時，必須設定此樣式。|
|BS_PUSHBUTTON|建立命令按鈕，在使用者按一下按鈕時，將 BN_CLICKED 通知傳送至主控視窗。|
|BS_RADIOBUTTON|建立具有兩個狀態的選項按鈕： BST_CHECKED 和 BST_UNCHECKED。 選項按鈕通常用於群組中，每個群組一次最多隻能有一個已核取的選項。 按一下按鈕，就會將 BN_CLICKED 通知傳送至 [擁有者] 視窗，但不會自動變更群組中任何按鈕的狀態。 根據預設，相關聯的文字會顯示在選項按鈕的右邊。 若要在選項按鈕左邊顯示文字，請使用 BS_LEFTTEXT 或 BS_RIGHTBUTTON 樣式。|
|BS_SPLITBUTTON|建立分割按鈕。 分割按鈕是 Windows Vista 特定的命令按鈕，其中包含下拉式箭號旁的按鈕。 當您按一下按鈕時，就會執行預設命令。 當您按一下下拉箭號時，就會出現其他命令的功能表。|
|BS_USERBUTTON|已過時，但提供的目的是為了與 Windows 的16位版本相容。 以 Win32 為基礎的應用程式應該改用 BS_OWNERDRAW。|

### <a name="radio-button-and-check-box-styles"></a>選項按鈕和核取方塊樣式

下表列出選項按鈕和核取方塊的特定樣式。 所有其他按鈕類型都會忽略這些樣式。 您可以選擇性地選擇下列一或多個動作。

|Style|描述|
|-----------|-----------------|
|BS_LEFTTEXT|結合選項按鈕或核取方塊樣式時，文字會出現在選項按鈕或核取方塊的左邊。|
|BS_RIGHTBUTTON|結合選項按鈕或核取方塊樣式時，文字會出現在選項按鈕或核取方塊的左邊。 此樣式等同于 BS_LEFTTEXT 樣式。|
|BS_PUSHLIKE|使核取方塊或選項按鈕的外觀和行為類似于命令按鈕。 當其狀態為 BST_CHECKED、已按下 或 暗灰色 BST_INDETERMINATE 時，按鈕會顯示為 已按下，並在其狀態為 已 BST_UNCHECKED|

### <a name="button-text-alignment-styles"></a>按鈕文字對齊樣式

下表列出水準和垂直文字對齊選項。 您可以選擇性地選擇下列其中一項。

|Style|描述|
|-----------|-----------------|
|BS_LEFT|靠左對齊按鈕矩形中的文字。 不過，如果按鈕是沒有 BS_RIGHTBUTTON 樣式的核取方塊或選項按鈕，則文字會靠左對齊核取方塊或選項按鈕的右邊。|
|BS_RIGHT|靠右對齊按鈕矩形中的文字。 不過，如果按鈕是沒有 BS_RIGHTBUTTON 樣式的核取方塊或選項按鈕，則文字會靠右對齊核取方塊或選項按鈕的右邊。|
|BS_CENTER|在按鈕矩形中將文字置中水準置中。|
|BS_TOP|將文字放在按鈕矩形的頂端。|
|BS_BOTTOM|將文字放在按鈕矩形的底部。|
|BS_VCENTER|在按鈕矩形中垂直置中文字。|

### <a name="button-content-options"></a>按鈕內容選項

下表列出的選項會指出按鈕中顯示的內容。 只顯示文字的按鈕類型會忽略這些樣式。 您可以選擇性地選擇下列其中一項。

|Style|描述|
|-----------|-----------------|
|BS_BITMAP|指定按鈕顯示點陣圖。|
|BS_ICON|指定按鈕顯示圖示。|
|BS_TEXT|指定按鈕顯示文字。|

### <a name="other-button-options"></a>其他按鈕選項

下表列出可用於任何按鈕類型的其他選項。 您可以選擇性地選擇下列一或多個動作。

|Style|描述|
|-----------|-----------------|
|BS_FLAT|指定按鈕為二維，而且不以預設陰影繪製，以建立3d 影像。|
|BS_MULTILINE|如果文字字串太長而無法放入按鈕矩形中的單行，請將按鈕文字包裝成多行。|
|BS_NOTIFY|啟用按鈕，將 BN_DBLCLK、BN_KILLFOCUS 和 BN_SETFOCUS 通知訊息傳送至其父視窗。 請注意，不論是否已指定此樣式，按鈕都會傳送 BN_CLICKED 通知。|

## <a name="combo-box-styles"></a>下拉式列示方塊樣式

MFC 中提供下列下拉式方塊樣式。 如需 Windows 中下拉式列示方塊樣式的詳細資訊，請參閱[下拉式方塊樣式（Windows）](/windows/win32/Controls/combo-box-styles)。

|Style|描述|
|-----------|-----------------|
|CBS_AUTOHSCROLL|當使用者在行尾輸入字元時，自動將編輯控制項中的文字向右滾動。 如果未設定此樣式，則只會允許符合矩形界限的文字。|
|CBS_DISABLENOSCROLL|當清單方塊未包含足夠的專案可供滾動時，清單方塊會顯示已停用的垂直捲動條。 若沒有這個樣式，在清單方塊中未包含足夠的項目時，捲軸會隱藏。|
|CBS_DROPDOWN|類似于 CBS_SIMPLE，不同之處在于除非使用者選取編輯控制項旁邊的圖示，否則不會顯示清單方塊。|
|CBS_DROPDOWNLIST|類似于 CBS_DROPDOWN，不同之處在于，編輯控制項會由顯示清單方塊中目前選取範圍的靜態文字專案所取代。|
|CBS_HASSTRINGS|主控描繪下拉式方塊包含由字串組成的專案。 下拉式方塊會維護記憶體和字串的指標，讓應用程式可以使用 `GetText` 成員函式來擷取特定項目的文字。|
|CBS_LOWERCASE|將選取範圍欄位和清單中的所有文字全部轉換成小寫。|
|CBS_NOINTEGRALHEIGHT|指定下拉式方塊的大小與應用程式建立下拉式方塊時所指定的大小完全相同。 一般而言，Windows 會調整下拉式方塊的大小，使下拉式方塊不會只顯示一部分的項目。|
|CBS_OEMCONVERT|在下拉式方塊編輯控制項中輸入的文字，會從 ANSI 字元集轉換成 OEM 字元集，然後再轉換回 ANSI。 如此可確保當應用程式呼叫 `AnsiToOem` Windows 函式將下拉式方塊中的 ANSI 字串轉換成 OEM 字元時，會進行適當的字元轉換。 此樣式是最適合用於包含檔案名稱的下拉式方塊，而且只適用於使用 CBS_SIMPLE 或 CBS_DROPDOWN 樣式建立的下拉式方塊。|
|CBS_OWNERDRAWFIXED|清單方塊的擁有者負責繪製其內容;清單方塊中的專案都具有相同的高度。|
|CBS_OWNERDRAWVARIABLE|清單方塊的擁有者負責繪製其內容;清單方塊中的專案是高度的變數。|

|SS_LEFT|指定一個簡單的矩形，並在矩形中顯示指定的文字排清。 文字會在顯示之前格式化。 超過行尾的文字會自動包裝至下一個排清行的開頭。|
|SS_LEFTNOWORDWRAP|指定一個簡單的矩形，並在矩形中顯示指定的文字排清。 索引標籤會展開，但不會包裝文字。 會裁剪超過行結尾的文字。|
|SS_NOPREFIX|除非指定了此樣式，否則 Windows 會將控制項文字中的任何 & 符號（&）字元解讀為快速鍵前置詞字元。 在此情況下，會移除連字號，而字串中的下一個字元會加上底線。 如果靜態控制項包含不需要這項功能的文字，可能會加入 SS_NOPREFIX。 這個靜態控制樣式可以包含在任何已定義的靜態控制項中。 您可以使用位 OR 運算子，將 SS_NOPREFIX 與其他樣式結合。 當可能包含連字號的檔案名或其他字串必須在對話方塊的靜態控制項中顯示時，最常使用此方式。|
|SS_NOTIFY|當使用者按一下或按兩下控制項時，將父視窗 STN_CLICKED、STN_DBLCLK、STN_DISABLE 和 STN_ENABLE 通知訊息傳送。|
|SS_OWNERDRAW|指定靜態控制項的擁有者負責繪製控制項。 每當需要繪製控制項時，擁有者視窗就會收到 WM_DRAWITEM 訊息。|
|SS_REALSIZEIMAGE|防止靜態圖示或點陣圖控制項（也就是具有 SS_ICON 或 SS_BITMAP 樣式的靜態控制項）在載入或繪製時重新調整大小。 如果圖示或點陣圖大於目的區域，則會裁剪影像。|
|SS_RIGHT|指定一個簡單的矩形，並在矩形中顯示指定的文字右下方。 文字會在顯示之前格式化。 超過行尾的文字會自動換行到下一個右邊的垂直線開頭。|
|SS_RIGHTJUST|指定具有 SS_BITMAP 或 SS_ICON 樣式之靜態控制項的右下角，會在調整控制項大小時維持固定狀態。 只有左上方和左側會調整以容納新的點陣圖或圖示。|
|SS_SIMPLE|指定一個簡單的矩形，並在矩形中顯示一行文字排清。 無法以任何方式縮短或更改文字行。 （控制項的父視窗或對話方塊不能處理 WM_CTLCOLOR 訊息）。|
|SS_SUNKEN|在靜態控制項周圍繪製半凹陷的框線。|
|SS_USERITEM|指定使用者定義的專案。|
|SS_WHITEFRAME|指定具有與視窗背景相同色彩繪製之框架的方塊。 預設為白色。|
|SS_WHITERECT|指定以用來填滿視窗背景的色彩填滿的矩形。 預設為白色。|
|SS_WORDELLIPSIS|截斷不符合並加入省略號的文字。|

## <a name="window-styles"></a>視窗樣式

視窗樣式適用于[CWnd 類別](../../mfc/reference/cwnd-class.md)物件。 在[CWnd：： Create](../../mfc/reference/cwnd-class.md#create)或[Cwnd：： CreateEx](../../mfc/reference/cwnd-class.md#createex)的*dwStyle*參數中指定樣式的組合。 如需 Windows 視窗樣式的詳細資訊，請參閱[視窗樣式（Windows）](/windows/win32/winmsg/window-styles)。

|Style|描述|
|-----------|-----------------|
|WS_BORDER|建立具有框線的視窗。|
|WS_CAPTION|建立具有標題列的視窗（隱含 WS_BORDER 樣式）。 不能與 WS_DLGFRAME 樣式一起使用。|
|WS_CHILD|建立子視窗。 不能與 WS_POPUP 樣式一起使用。|
|WS_CHILDWINDOW|與 WS_CHILD 樣式相同。|
|WS_CLIPCHILDREN|在父視窗內繪圖時，應排除子視窗佔用的區域。 當您建立父視窗時使用。|
|WS_CLIPSIBLINGS|裁剪相對於彼此的子視窗;也就是說，當特定的子視窗收到繪製訊息時，WS_CLIPSIBLINGS 樣式會將所有其他重迭的子視窗，裁剪為要更新之子視窗的區域。 （如果未指定 WS_CLIPSIBLINGS，而子視窗重迭，則當您在子視窗的工作區中繪製時，可以在相鄰子視窗的工作區中繪製。）僅用於 WS_CHILD 樣式。|
|WS_DISABLED|建立一個一開始即停用的視窗。|
|WS_DLGFRAME|建立具有雙框線但沒有標題的視窗。|
|WS_GROUP|指定控制項群組的第一個控制項，使用者可以在其中使用方向鍵從某個控制項移到下一個控制項。 在第一個控制項屬於相同群組之後，以 WS_GROUP 樣式 FALSE 定義的所有控制項。 下一個具有 WS_GROUP 樣式的控制項會啟動下一個群組（也就是一個群組結束于下一個開始的位置）。|
|WS_HSCROLL|建立具有水準捲軸的視窗。|
|WS_ICONIC|建立一開始最小化的視窗。 與 WS_MINIMIZE 樣式相同。|
|WS_MAXIMIZE|建立大小上限的視窗。|
|WS_MAXIMIZEBOX|建立具有 [最大化] 按鈕的視窗。|
|WS_MINIMIZE|建立一開始最小化的視窗。 僅用於 WS_OVERLAPPED 樣式。|
|WS_MINIMIZEBOX|建立具有 [最小化] 按鈕的視窗。|
|WS_OVERLAPPED|建立重迭視窗。 重迭的視窗通常會有標題和框線。|
|WS_OVERLAPPEDWINDOW|建立具有 WS_OVERLAPPED、WS_CAPTION、WS_SYSMENU、WS_THICKFRAME、WS_MINIMIZEBOX 和 WS_MAXIMIZEBOX 樣式的重迭視窗。|
|WS_POPUP|建立快顯視窗。 不能與 WS_CHILD 樣式一起使用。|
|WS_POPUPWINDOW|建立具有 WS_BORDER、WS_POPUP 和 WS_SYSMENU 樣式的快顯視窗。 WS_CAPTION 樣式必須與 WS_POPUPWINDOW 樣式結合，讓控制項功能表顯示。|
|WS_SIZEBOX|建立具有調整大小框線的視窗。 與 WS_THICKFRAME 樣式相同。|
|WS_SYSMENU|建立視窗，其標題列中有 [控制] 功能表框。 僅用於具有標題列的 windows。|
|WS_TABSTOP|指定使用者可以使用 TAB 鍵移動的任意數目控制項之一。 TAB 鍵會將使用者移至 [WS_TABSTOP] 樣式所指定的下一個控制項。|
|WS_THICKFRAME|使用可用於調整視窗大小的寬框架來建立視窗。|
|WS_TILED|建立重迭視窗。 重迭的視窗有標題列和框線。 與 WS_OVERLAPPED 樣式相同。|
|WS_TILEDWINDOW|建立具有 WS_OVERLAPPED、WS_CAPTION、WS_SYSMENU、WS_THICKFRAME、WS_MINIMIZEBOX 和 WS_MAXIMIZEBOX 樣式的重迭視窗。 與 WS_OVERLAPPEDWINDOW 樣式相同。|
|WS_VISIBLE|建立一個一開始即可見的視窗。|
|WS_VSCROLL|建立具有垂直捲動條的視窗。|

## <a name="extended-window-styles"></a>擴充的視窗樣式

延伸視窗樣式適用于[CWnd 類別](../../mfc/reference/cwnd-class.md)物件。 在[CWnd：： CreateEx](../../mfc/reference/cwnd-class.md#createex)的*dwExStyle*參數中指定樣式的組合。 如需 Windows 延伸視窗樣式的詳細資訊，請參閱[擴充視窗樣式（Windows）](/windows/win32/winmsg/extended-window-styles)。

|Style|描述|
|-----------|-----------------|
|WS_EX_ACCEPTFILES|指定使用此樣式所建立的視窗接受拖放檔案。|
|WS_EX_APPWINDOW|當視窗可見時，將最上層視窗強制到工作列上。|
|WS_EX_CLIENTEDGE|指定視窗具有3D 外觀（也就是具有凹陷邊緣的框線）。|
|WS_EX_CONTEXTHELP|在視窗的標題列中包含問號。 當使用者按一下問號時，游標會變成具有指標的問號。 如果使用者按一下子視窗，子視窗會收到 WM_HELP 訊息。|
|WS_EX_CONTROLPARENT|允許使用者在視窗的子視窗中，使用 TAB 鍵進行流覽。|
|WS_EX_DLGMODALFRAME|當您在*dwStyle*參數中指定 WS_CAPTION 樣式旗標時，使用標題列來指定具有雙框線的視窗（選擇性）。|
|WS_EX_LAYERED|視窗是[分層視窗](/windows/win32/winmsg/window-features)。 如果視窗的[類別樣式](/windows/win32/winmsg/about-window-classes)為 CS_OWNDC 或 CS_CLASSDC，則無法使用此樣式。 不過，Microsoft Windows 8 確實支援子視窗的 WS_EX_LAYERED 樣式，其中先前的 Windows 版本僅支援最上層視窗。|
|WS_EX_LEFT|提供視窗泛型靠左對齊的屬性。 這是預設值。|
|WS_EX_LEFTSCROLLBAR|將垂直捲動條放在工作區的左邊。|
|WS_EX_LTRREADING|使用由左到右的讀取順序屬性來顯示視窗文字。 這是預設值。|
|WS_EX_MDICHILD|建立 MDI 子視窗。|
|WS_EX_NOPARENTNOTIFY|指定在建立或終結子視窗時，使用這個樣式所建立的子視窗不會將 WM_PARENTNOTIFY 訊息傳送至其父視窗。|
|WS_EX_OVERLAPPEDWINDOW|結合 WS_EX_CLIENTEDGE 和 WS_EX_WINDOWEDGE 樣式|
|WS_EX_PALETTEWINDOW|結合 WS_EX_WINDOWEDGE 和 WS_EX_TOPMOST 樣式。|
|WS_EX_RIGHT|提供視窗的一般靠右對齊屬性。 這取決於視窗類別。|
|WS_EX_RIGHTSCROLLBAR|將垂直捲動條（如果有的話）放在工作區的右邊。 這是預設值。|
|WS_EX_RTLREADING|使用由右至左的讀取順序屬性來顯示視窗文字。|
|WS_EX_STATICEDGE|建立具有三維框線樣式的視窗，其適用于不接受使用者輸入的專案。|
|WS_EX_TOOLWINDOW|建立工具視窗，這是要當做浮動工具列使用的視窗。 工具視窗的標題列較一般標題列短，而且使用小字型繪製視窗標題。 工具視窗不會在工作列中顯示，當使用者按下 ALT+TAB 時也不會在出現的視窗中顯示。|
|WS_EX_TOPMOST|指定使用此樣式建立的視窗應該放在所有 nontopmost 視窗上方，並在視窗停用時保持在其上方。 應用程式可以使用 `SetWindowPos` 成員函式來加入或移除此屬性。|
|WS_EX_TRANSPARENT|指定使用這個樣式所建立的視窗是透明的。 也就是說，在這個視窗下的任何視窗不會被遮蔽。 使用此樣式所建立的視窗，只有在所有已更新的同輩視窗都已更新之後，才會收到 WM_PAINT 訊息。|
|WS_EX_WINDOWEDGE|指定視窗具有凸起邊緣的框線。|
