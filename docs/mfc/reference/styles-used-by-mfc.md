# <a name="styles-used-by-mfc"></a>MFC 使用的樣式

您可以使用下列樣式旗標來指定視窗或控制項的外觀和行為，當您建立相對應 MFC 物件。 在大部分情況下，在中設定這些樣式*cheaderctrl:: Create*類別的參數`Create`函式。

## <a name="button-styles"></a> 按鈕樣式

按鈕樣式套用至[CButton 類別](../../mfc/reference/cbutton-class.md)物件，例如選項按鈕、 核取方塊和按鈕。 指定的組合中的樣式*cheaderctrl:: Create*的參數[CButton::Create](../../mfc/reference/cbutton-class.md#create)。 如需有關在 Windows 中的按鈕樣式的詳細資訊，請參閱 <<c0> [ 按鈕樣式 (Windows)](/windows/desktop/Controls/button-styles)。

### <a name="button-types"></a>按鈕類型

下表列出按鈕類型。 您可以選擇性地選擇下列其中一個項目。 如果您未指定的按鈕類型，則預設為 BS_PUSHBUTTON。

|類型|描述|
|----------|-----------------|
|BS_3STATE|建立具有三種狀態的核取方塊按鈕： BST_CHECKED、 BST_INDETERMINATE 和 BST_UNCHECKED。 按一下這個按鈕會 BN_CLICKED 通知傳送給主控視窗，但不會變更按鈕的狀態。 根據預設，相關聯的文字會顯示右邊的核取方塊。 若要顯示文字左邊的核取方塊，請使用 BS_LEFTTEXT 或 BS_RIGHTBUTTON 樣式。|
|BS_AUTO3STATE|建立具有三種狀態的核取方塊按鈕： BST_CHECKED、 BST_INDETERMINATE 和 BST_UNCHECKED。 按一下這個按鈕會 BN_CLICKED 通知傳送給主控視窗，並變更按鈕的狀態。 按鈕狀態 BST_CHECKED、 BST_INDETERMINATE 和 BST_UNCHECKED 順序循環。 根據預設，相關聯的文字會顯示右邊的核取方塊。 若要顯示文字左邊的核取方塊，請使用 BS_LEFTTEXT 或 BS_RIGHTBUTTON 樣式。|
|BS_AUTOCHECKBOX|建立具有兩個狀態的核取方塊按鈕： BST_CHECKED 和 BST_UNCHECKED。 按一下這個按鈕會 BN_CLICKED 通知傳送給主控視窗，並變更按鈕的狀態。 根據預設，相關聯的文字會顯示右邊的核取方塊。 若要顯示文字左邊的核取方塊，請使用 BS_LEFTTEXT 或 BS_RIGHTBUTTON 樣式。|
|BS_AUTORADIOBUTTON|建立具有兩種狀態的選項按鈕： BST_CHECKED 和 BST_UNCHECKED。 選項按鈕通常用在群組中，而其中一個選取的選項，一次最多每個群組。 按一下這個按鈕會 BN_CLICKED 通知傳送給主控視窗、 按下的選項按鈕的狀態設定為 BST_CHECKED，和設定 BST_UNCHECKED 按鈕群組中的所有其他選項按鈕的狀態。 根據預設，相關聯的文字會顯示選項按鈕的右邊。 左邊的選項按鈕的顯示文字，請使用 BS_LEFTTEXT 或 BS_RIGHTBUTTON 樣式。|
|BS_CHECKBOX|建立具有兩個狀態的核取方塊按鈕： BST_CHECKED 和 BST_UNCHECKED。 按一下這個按鈕會 BN_CLICKED 通知傳送給主控視窗，但不會變更按鈕的狀態。 根據預設，相關聯的文字會顯示右邊的核取方塊。 若要顯示文字左邊的核取方塊，請使用 BS_LEFTTEXT 或 BS_RIGHTBUTTON 樣式。|
|BS_COMMANDLINK|建立命令連結按鈕。 命令連結按鈕是專屬 Windows Vista 的主要文字下方，記下主要文字左邊顯示一個綠色箭號的命令按鈕。 您可以設定附註文字使用[CButton::SetNote](../../mfc/reference/cbutton-class.md#setnote)。|
|BS_DEFCOMMANDLINK|建立命令連結按鈕。 命令連結按鈕是專屬 Windows Vista 的主要文字下方，記下主要文字左邊顯示一個綠色箭號的命令按鈕。 您可以設定附註文字使用[CButton::SetNote](../../mfc/reference/cbutton-class.md#setnote)。 如果按鈕在對話方塊中，按下 ENTER 鍵傳送 BN_CLICKED 通知給對話方塊即使按鈕沒有輸入的焦點。|
|BS_DEFPUSHBUTTON|建立具有沉重黑色框線的命令按鈕。 如果按鈕在對話方塊中，按下 ENTER 鍵傳送 BN_CLICKED 通知給對話方塊即使按鈕沒有輸入的焦點。|
|BS_DEFSPLITBUTTON|建立分割按鈕。 分割按鈕是專屬 Windows Vista 包含相鄰的下拉式箭號按鈕的命令按鈕。 當您按一下按鈕時，會執行預設命令。 當您按一下下拉箭號時，則會顯示其他命令的功能表。 如果分割按鈕是在對話方塊中，按下 ENTER 鍵 BN_CLICKED 將通知傳送給對話方塊即使按鈕沒有輸入的焦點|
|BS_GROUPBOX|建立其他按鈕可以群組的矩形。 此樣式相關聯的文字會顯示在矩形的左上角。|
|BS_OWNERDRAW|建立主控描繪按鈕。 這個架構會呼叫`DrawItem`當按鈕的視覺外觀的方法已經變更。 當您使用時，就必須設定此樣式`CBitmapButton`類別。|
|BS_PUSHBUTTON|建立使用者按一下按鈕時，傳送 BN_CLICKED 通知給主控視窗的命令按鈕。|
|BS_RADIOBUTTON|建立具有兩種狀態的選項按鈕： BST_CHECKED 和 BST_UNCHECKED。 選項按鈕通常用在群組中，而其中一個選取的選項，一次最多每個群組。 按一下這個按鈕會 BN_CLICKED 通知傳送給主控視窗，但不會自動變更群組中的任何按鈕的狀態。 根據預設，相關聯的文字會顯示選項按鈕的右邊。 左邊的選項按鈕的顯示文字，請使用 BS_LEFTTEXT 或 BS_RIGHTBUTTON 樣式。|
|BS_SPLITBUTTON|建立分割按鈕。 分割按鈕是專屬 Windows Vista 包含相鄰的下拉式箭號按鈕的命令按鈕。 當您按一下按鈕時，會執行預設命令。 當您按一下下拉箭號時，則會顯示其他命令的功能表。|
|BS_USERBUTTON|已淘汰，但會提供給與 16 位元版本的 Windows 相容性。 Win32 應用程式應該改用 BS_OWNERDRAW。|

### <a name="radio-button-and-check-box-styles"></a>選項按鈕和核取方塊樣式

下表列出專屬於選項按鈕和核取方塊的樣式。 這些樣式會忽略所有其他的按鈕類型。 您可以選擇一或多個項目。

|樣式|描述|
|-----------|-----------------|
|BS_LEFTTEXT|與選項按鈕或核取方塊樣式結合時，文字會出現在左邊的選項按鈕或核取方塊。|
|BS_RIGHTBUTTON|與選項按鈕或核取方塊樣式結合時，文字會出現在左邊的選項按鈕或核取方塊。 這個樣式等同於 BS_LEFTTEXT 樣式。|
|BS_PUSHLIKE|可讓核取方塊或選項按鈕的外觀和行為類似的命令按鈕。 當其狀態是 BST_CHECKED，按下並呈暗灰色，其狀態時 BST_INDETERMINATE，並釋放其狀態為 BST_UNCHECKED 時，會出現已按下按鈕。|

### <a name="button-text-alignment-styles"></a>按鈕文字的對齊樣式

下表列出水平和垂直文字對齊選項。 您可以選擇性地選擇下列其中一個項目。

|樣式|描述|
|-----------|-----------------|
|BS_LEFT|將按鈕矩形中的文字靠左對齊。 不過，如果按鈕 核取方塊或選項按鈕，並沒有 BS_RIGHTBUTTON 樣式，則文字會保持核取方塊或選項按鈕的右邊對齊。|
|BS_RIGHT|將按鈕矩形中的文字靠右對齊。 不過，如果按鈕的核取方塊或選項按鈕，並沒有 BS_RIGHTBUTTON 樣式，文字會靠右對齊核取方塊或選項按鈕的右邊。|
|BS_CENTER|將按鈕矩形中的水平文字置中。|
|BS_TOP|將文字放在按鈕矩形的頂端。|
|BS_BOTTOM|將文字放在按鈕矩形的底部。|
|BS_VCENTER|將按鈕矩形中，垂直文字置中。|

### <a name="button-content-options"></a>按鈕內容選項

下表列出指示在按鈕中顯示的內容的選項。 只顯示文字的按鈕類型會忽略這些樣式。 您可以選擇性地選擇下列其中一個項目。

|樣式|描述|
|-----------|-----------------|
|BS_BITMAP|指定按鈕顯示點陣圖。|
|BS_ICON|指定按鈕顯示的圖示。|
|BS_TEXT|指定按鈕顯示文字。|

### <a name="other-button-options"></a>其他按鈕選項

下表列出您可以搭配任何按鈕類型使用的其他選項。 您可以選擇一或多個項目。

|樣式|描述|
|-----------|-----------------|
|BS_FLAT|指定按鈕是二維，不使用建立三維影像的預設陰影繪製。|
|BS_MULTILINE|包裝多行按鈕文字，如果文字字串太長而無法容納在按鈕矩形中，每一行上。|
|BS_NOTIFY|可讓按鈕 BN_DBLCLK、 BN_KILLFOCUS 和 BN_SETFOCUS 通知訊息傳送至其父視窗。 請注意按鈕傳送 BN_CLICKED 通知，不論是否指定此樣式。|

## <a name="combo-box-styles"></a>  下拉式方塊樣式

MFC 中提供下列下拉式方塊樣式。 如需有關在 Windows 中的下拉式方塊樣式的詳細資訊，請參閱 <<c0> [ 下拉式方塊樣式 (Windows)](/windows/desktop/Controls/combo-box-styles)。

|樣式|描述|
|-----------|-----------------|
|CBS_AUTOHSCROLL|當使用者在行結尾處輸入字元時，會自動捲動右邊的編輯控制項中的文字。 如果未設定此樣式，則只會允許符合矩形界限的文字。|
|CBS_DISABLENOSCROLL|清單方塊會顯示已停用的垂直捲軸當清單方塊中未包含足夠的項目捲動。 若沒有這個樣式，在清單方塊中未包含足夠的項目時，捲軸會隱藏。|
|CBS_DROPDOWN|類似於 CBS_SIMPLE，不過除非使用者選取編輯控制項旁的圖示，將不顯示清單方塊。|
|CBS_DROPDOWNLIST|類似於 CBS_DROPDOWN，不過編輯控制項會由取代的靜態文字項目，會顯示目前的選取範圍的清單方塊中。|
|CBS_HASSTRINGS|主控描繪下拉式方塊包含由字串組成的項目。 下拉式方塊會維護記憶體和字串的指標，讓應用程式可以使用 `GetText` 成員函式來擷取特定項目的文字。|
|CBS_LOWERCASE|將轉換成小寫的選取項目欄位和清單中的所有文字。|
|CBS_NOINTEGRALHEIGHT|指定下拉式方塊的大小是建立下拉式方塊時，應用程式所指定的大小相同。 一般而言，Windows 會調整下拉式方塊的大小，使下拉式方塊不會只顯示一部分的項目。|
|CBS_OEMCONVERT|在下拉式方塊編輯控制項中輸入的文字會從 ANSI 字元集轉換，然後回 ANSI 和 OEM 字元集。 如此可確保當應用程式呼叫 `AnsiToOem` Windows 函式將下拉式方塊中的 ANSI 字串轉換成 OEM 字元時，會進行適當的字元轉換。 此樣式是最適合用於包含檔案名稱的下拉式方塊，而且只適用於建立具有 CBS_SIMPLE 或 CBS_DROPDOWN 樣式的下拉式方塊。|
|CBS_OWNERDRAWFIXED|清單方塊的擁有者會負責繪製其內容;在清單方塊中的項目都有相同的高度。|
|CBS_OWNERDRAWVARIABLE|清單方塊的擁有者會負責繪製其內容;在清單方塊中的項目高度是可變的。|
|CBS_SIMPLE|清單方塊會顯示在所有的時間。 編輯控制項會顯示清單方塊中目前的選取範圍。|
|CBS_SORT|自動排序輸入至清單方塊中的字串。|

|SS_NOPREFIX|除非指定此樣式時，Windows 將解譯任何連字號 (&) 是對應鍵前置字元的控制項的文字中的字元。 在此情況下，已移除連字號，並在字串中的下一個字元會加上底線。 如果靜態控制項，以包含文字不需要這項功能時，則可能會新增 SS_NOPREFIX。 這個靜態控制項樣式可能包含任何已定義的靜態控制項。 您可以具有其他樣式結合 SS_NOPREFIX，方法是使用位元的 OR 運算子。 這最常使用時要顯示在對話方塊中的靜態控制項中需要的檔案名稱或其他可能包含連字號的字串。|
|呼叫 SS_NOTIFY|當使用者按一下或按兩下控制項時傳送的父視窗 STN_CLICKED、 STN_DBLCLK、 STN_DISABLE 和 STN_ENABLE 通知訊息。|
|SS_OWNERDRAW|指定靜態控制項的擁有者會負責繪製控制項。 每當控制項需要繪製主控視窗會收到 WM_DRAWITEM 訊息。|
|SS_REALSIZEIMAGE|正在調整大小是載入或繪製，所以可避免靜態圖示或點陣圖控制項 （也就是靜態控制項具有 SS_ICON 或 SS_BITMAP 樣式的）。 如果圖示或點陣圖大於目的區域，則會裁剪影像。|
|SS_RIGHT|指定一個簡單的矩形，並顯示指定的文字的矩形中排清權限。 文字格式顯示。 會延伸超過行尾的字組自動包裝至下一步 的清除權限列的開頭。|
|SS_RIGHTJUST|指定靜態 SS_BITMAP 或 SS_ICON 樣式控制項的右下角保持固定時調整控制項大小。 僅線上方和左側會調整，以容納新的點陣圖或圖示。|
|SS_SIMPLE|指定一個簡單的矩形，並顯示 單行文字的矩形中排清左。 無法縮短的一行文字，或以任何方式改變。 （控制項的父視窗或對話方塊方塊必須處理 WM_CTLCOLOR 訊息）。|
|SS_SUNKEN|繪製靜態控制項半下凹框線。|
|SS_USERITEM|指定使用者定義的項目。|
|SS_WHITEFRAME|指定方塊有使用相同的視窗背景色彩繪製框架。 預設為白色。|
|SS_WHITERECT|指定用來填滿視窗背景的色彩填滿的矩形。 預設為白色。|
|SS_WORDELLIPSIS|不符合，並將省略符號的文字就會截斷。|

## <a name="window-styles"></a> 視窗樣式

視窗樣式套用至[CWnd 類別](../../mfc/reference/cwnd-class.md)物件。 指定的組合中的樣式*cheaderctrl:: Create*的參數[cwnd:: Create](../../mfc/reference/cwnd-class.md#create)或是[CWnd::CreateEx](../../mfc/reference/cwnd-class.md#createex)。 如需有關在 Windows 中的視窗樣式的詳細資訊，請參閱 <<c0> [ 的視窗樣式 (Windows)](/windows/desktop/winmsg/window-styles)。

|樣式|描述|
|-----------|-----------------|
|WS_BORDER|建立具有框線的視窗。|
|WS_CAPTION|建立具有標題列 （意指 WS_BORDER 樣式） 的視窗。 無法搭配 WS_DLGFRAME 樣式。|
|WS_CHILD|建立子視窗。 無法搭配 WS_POPUP 樣式。|
|WS_CHILDWINDOW|WS_CHILD 樣式相同。|
|WS_CLIPCHILDREN|在父視窗內繪圖時，應排除子視窗佔用的區域。 當您建立父視窗時使用。|
|WS_CLIPSIBLINGS|相對於其他; 裁剪子視窗也就是當特定子視窗收到繪製訊息時，WS_CLIPSIBLINGS 樣式會裁剪子視窗，更新的區域的所有其他重疊的子視窗。 （如果未指定 WS_CLIPSIBLINGS 子視窗重疊，工作區之子視窗內繪圖時，很可能要在其中繪製在鄰近子視窗的工作區。）WS_CHILD 樣式只搭配使用。|
|WS_DISABLED|建立一個一開始即停用的視窗。|
|WS_DLGFRAME|建立具有雙框線，但沒有標題的視窗。|
|WS_GROUP|指定在其中，使用者可以移動從一個控制項到下一步 使用方向鍵的控制項群組的第一個控制項。 WS_GROUP 樣式 FALSE 定義之後的第一個控制項的所有控制項都屬於相同的群組。 WS_GROUP 樣式的下一個控制項開始下一步 的群組 （也就是一個群組結束下一步 開始的位置）。|
|WS_HSCROLL|建立有水平捲軸的視窗。|
|WS_ICONIC|建立一開始最小化的視窗。 WS_MINIMIZE 樣式相同。|
|WS_MAXIMIZE|建立視窗的大小上限。|
|WS_MAXIMIZEBOX|建立有最大化 按鈕的視窗。|
|WS_MINIMIZE|建立一開始最小化的視窗。 WS_OVERLAPPED 樣式只搭配使用。|
|WS_MINIMIZEBOX|建立具有最小化按鈕的視窗。|
|WS_OVERLAPPED|建立重疊的視窗。 重疊的視窗通常具有標題和框線。|
|WS_OVERLAPPEDWINDOW|建立具有 WS_OVERLAPPED、 WS_CAPTION、 WS_SYSMENU、 WS_THICKFRAME、 WS_MINIMIZEBOX 和 WS_MAXIMIZEBOX 樣式的重疊的視窗。|
|WS_POPUP|建立快顯視窗。 無法搭配 WS_CHILD 樣式。|
|WS_POPUPWINDOW|建立快顯視窗具有 WS_BORDER、 WS_POPUP 和 WS_SYSMENU 樣式。 WS_CAPTION 樣式必須結合 WS_POPUPWINDOW 樣式，以顯示 [控制] 功能表。|
|WS_SIZEBOX|建立具有縮放邊框的視窗。 WS_THICKFRAME 樣式相同。|
|WS_SYSMENU|建立具有其標題列中的控制功能表方塊的視窗。 使用此選項僅適用於具有標題列的 windows。|
|WS_TABSTOP|指定任意數目的控制項，讓使用者可以使用 TAB 鍵移動的其中一個。 TAB 鍵會將使用者移至下一個 WS_TABSTOP 樣式所指定的控制項。|
|WS_THICKFRAME|建立具有可用來調整視窗大小粗框架視窗。|
|WS_TILED|建立重疊的視窗。 重疊的視窗具有標題列和框線。 WS_OVERLAPPED 樣式相同。|
|WS_TILEDWINDOW|建立具有 WS_OVERLAPPED、 WS_CAPTION、 WS_SYSMENU、 WS_THICKFRAME、 WS_MINIMIZEBOX 和 WS_MAXIMIZEBOX 樣式的重疊的視窗。 WS_OVERLAPPEDWINDOW 樣式相同。|
|WS_VISIBLE|建立一個一開始即可見的視窗。|
|WS_VSCROLL|建立有垂直捲軸的視窗。|

## <a name="extended-window-styles"></a> 延伸的視窗樣式

延伸的視窗樣式套用至[CWnd 類別](../../mfc/reference/cwnd-class.md)物件。 指定的組合中的樣式*dwExStyle*的參數[CWnd::CreateEx](../../mfc/reference/cwnd-class.md#createex)。 如需有關在 Windows 中的延伸的視窗樣式的詳細資訊，請參閱[延伸視窗樣式 (Windows)](/windows/desktop/winmsg/extended-window-styles)。

|樣式|描述|
|-----------|-----------------|
|WS_EX_ACCEPTFILES|指定以這個樣式的視窗接受拖放檔案。|
|WS_EX_APPWINDOW|當視窗為可見時，會強制放在工作列上的最上層視窗。|
|WS_EX_CLIENTEDGE|指定視窗有 3D 外觀 — 也就是下凹邊緣的框線。|
|WS_EX_CONTEXTHELP|在視窗的標題列包含問號。 當使用者按一下問號，游標會變為有指標的問號。 如果使用者再按一下子視窗，子視窗會收到 WM_HELP 訊息。|
|WS_EX_CONTROLPARENT|可讓使用者使用 TAB 鍵瀏覽不同的子視窗的視窗。|
|WS_EX_DLGMODALFRAME|指定的視窗標題列中的 WS_CAPTION 樣式旗標的指定時，（選擇性） 可能建立有雙框線*cheaderctrl:: Create*參數。|
|WS_EX_LAYERED|視窗[分層視窗](/windows/desktop/winmsg/window-features)。 如果視窗有，就無法使用這個樣式[類別樣式](/windows/desktop/winmsg/about-window-classes)CS_OWNDC 或 CS_CLASSDC。 不過，Microsoft Windows 8 的子視窗，其中 Windows 上的舊版支援僅適用於最上層的 windows 支援 WS_EX_LAYERED 樣式。|
|WS_EX_LEFT|給予視窗泛型靠左對齊屬性。 這是預設值。|
|WS_EX_LEFTSCROLLBAR|工作區左邊放置垂直捲軸。|
|WS_EX_LTRREADING|顯示視窗文字使用左到右讀取順序屬性。 這是預設值。|
|WS_EX_MDICHILD|建立 MDI 子視窗。|
|WS_EX_NOPARENTNOTIFY|指定的子視窗，以這個樣式不會傳送 WM_PARENTNOTIFY 訊息到其父視窗時建立或終結子視窗。|
|WS_EX_OVERLAPPEDWINDOW|結合 WS_EX_CLIENTEDGE 與 WS_EX_WINDOWEDGE 樣式|
|WS_EX_PALETTEWINDOW|結合 WS_EX_WINDOWEDGE 和 WS_EX_TOPMOST 樣式。|
|WS_EX_RIGHT|給予視窗泛型靠右對齊屬性。 這取決於視窗類別。|
|WS_EX_RIGHTSCROLLBAR|向工作區的放置垂直捲軸 （如果有的話）。 這是預設值。|
|WS_EX_RTLREADING|顯示視窗文字使用由右至左讀取順序屬性。|
|WS_EX_STATICEDGE|建立具有適用於不接受使用者輸入的項目有 3d 框線樣式的視窗。|
|WS_EX_TOOLWINDOW|建立工具視窗，也就是可以用來當做浮動工具列的視窗。 工具視窗的標題列較一般標題列短，而且使用小字型繪製視窗標題。 工具視窗不會在工作列中顯示，當使用者按下 ALT+TAB 時也不會在出現的視窗中顯示。|
|WS_EX_TOPMOST|指定以這個樣式的視窗應該放以上所有產生的 windows，且視窗已停用時，即使保持。 應用程式可以使用 `SetWindowPos` 成員函式來加入或移除此屬性。|
|WS_EX_TRANSPARENT|指定以這個樣式的視窗為透明。 也就是說，在這個視窗下的任何視窗不會被遮蔽。 只有在其下的所有同層級視窗更新之後，才建立以這個樣式的視窗會收到 WM_PAINT 訊息。|
|WS_EX_WINDOWEDGE|指定視窗具有凸起邊緣的框線。|
